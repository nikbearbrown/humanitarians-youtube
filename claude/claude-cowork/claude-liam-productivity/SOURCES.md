## Claims carried (verified true against the source + book FACTCHECK.md)
- The productivity plugin is the *generalist* — it manages the daily flow of tasks, meetings, communications, and priorities (specializes in the user, not a domain). ✓
- Capabilities: task management, meeting preparation, email triage, scheduling awareness, note synthesis. ✓
- Works immediately after installation; **no external connections required**. ✓
- Calendar connection is **optional**; the plugin is useful without it, just less time-aware. Standard authorization flow (connect → sign in → grant). ✓
- It is NOT a replacement for a project-management app or calendar — it's connective tissue. ✓
- Signature workflows: morning briefing, meeting preparation, email triage, weekly review, capture-without-breaking-flow. ✓
- It adapts to the user's own prioritization method (e.g. an urgency-importance / Eisenhower matrix) rather than imposing one. ✓
- It improves the more Claude knows about recurring commitments, projects, and patterns; global instructions provide persistent context across sessions. ✓
- Most people install it first and keep it permanently active as the foundation layer under domain plugins. ✓
