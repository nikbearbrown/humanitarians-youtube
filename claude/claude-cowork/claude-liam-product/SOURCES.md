## Claims carried (verified against source; evergreen)
- The product plugin brings product-management discipline to solo builders / people without a product team. ✓
- It produces: feature specs / PRDs, user stories, feedback (research) synthesis, roadmap/prioritization, release notes. ✓
- Specs define what/who/user-flow/edge-cases/acceptance criteria. ✓
- User stories decompose a big feature into pieces buildable in a focused session. ✓
- Feedback synthesis ranks patterns by frequency and severity; separates the loudest voice from the most common need. ✓
- Prioritization is articulation-forced ("why does one feature matter more than another"), not a fancy framework. ✓
- Release notes are written from the user's perspective (benefit, not implementation). ✓
- Decision documentation records trade-offs so future-you remembers the why. ✓
- Works from what you tell it even with no connected systems; connectors add usage/PM data. ✓ (connector *examples* kept fleeting only.)

## Judgment line (the reel's thesis, honest per DOUBLE-CHECK LAW)
