## Per-Video Loop — Step Sequence (per video)

1. Check `short/<slug>-short-cut.mp4` exists → SKIPPED(exists).
2. Run `python3 runtime/scripts/shorts.py <book>/youtube/<slug> --dry-run` to
   confirm ONDA check finds all 916 twins (zero flags expected after setup).
3. Run `python3 runtime/scripts/shorts.py <book>/youtube/<slug>` (straight) or
   `--cut BHTF` (beat cut). This writes `short/beat_sheet.json` and prints the
   render commands.
4. Run `python3 runtime/scripts/remotion_scenes.py <book>/youtube/<slug>/short`
   (FOREGROUND, `--concurrency=1`) for any ONDA-rewired beats.
5. Run `python3 runtime/scripts/compile.py <book>/youtube/<slug>/short --height 1920 --width 1080`
6. Probe duration: `ffprobe -v quiet -show_entries format=duration -of csv=p=0 <mp4>`.
   Fail loud if > 175s.
7. Visual QC: `ffmpeg -vf "fps=2" short/_qc/<slug>-%03d.png` + per-beat frames
   at 15/50/85%. Read all PNGs. Check SAFE916 8-point rubric. Log to
   `short/_qc/REPORT.md`. Zero BLOCKERs/MAJORs before done.
8. On success: log one line to `BUILD-LOG.md`.
